# SVN Studio 真人高频变更工作区 UI/UX 设计

| 项 | 内容 |
|----|------|
| 日期 | 2026-07-15 |
| 产品 | SVN Studio |
| 状态 | 已批准（用户授权后续设计决策自主推进） |
| 对应迭代 | Human UI Wave U5 |

## 1. 背景

SVN Studio 已完成 TortoiseSVN 功能对标，但当前主窗口仍以“功能可达”为主要组织方式。真实窗口在约 `1180 x 760` 下存在以下问题：

- 工作副本侧栏过窄，名称、路径和仓库 URL 互相争抢空间；
- 顶部模式、更多、工具和仓库摘要平铺在同一行，视觉层级弱；
- 变更区同时展示筛选、视图、搜索、列配置、刷新、远端检查和大量文件操作，按钮压缩为省略号；
- 未选择文件时，Diff 将空闲状态错误显示为“加载 diff…”；
- 变更选择、Diff 焦点和提交候选各自维护状态，用户难以确认“正在看什么、将提交什么”；
- 提交区常驻占据大量垂直空间，AI 操作与提交操作同级，主流程不清晰；
- 状态提示散落，加载、错误、成功和空态会引发布局跳动。

本设计把产品重新校准为给人高频使用的专业 macOS SVN 客户端。AI 继续保留完整能力，但只作为可选辅助，不占据基础版本控制工作流的主操作层级。

## 2. 长程范围与本阶段边界

完整 UI 长程目标分四个连续阶段推进：

1. **U5 高频变更工作区**：工作副本侧栏、模式栏、变更、Diff、提交；
2. **U6 核心模式统一**：历史、仓库浏览、分支、冲突；
3. **U7 辅助能力统一**：属性、锁、搁置、设置、弹窗、进度与错误反馈；
4. **U8 全局体验收口**：键盘流、无障碍、动效、性能、视觉一致性与真人任务验收。

本文只定义 U5 的可实现范围。完成 U5 不代表长程 UI 目标完成，后续阶段必须继续。

业务能力、SVN 命令语义、深链、命令面板和现有 Tortoise 对标能力不得删减。允许改变呈现位置和交互顺序，不允许以“简化 UI”为理由砍掉能力。

## 3. 设计原则

1. **任务优先**：先让用户明确当前工作副本、当前文件和待提交集合，再展示高级能力。
2. **一屏一主操作**：变更工作区只有“提交”使用强调样式；破坏性操作保持明确但不抢主层级。
3. **渐进披露**：高频操作直接可达，低频或需要参数的操作进入语义清楚的菜单或弹窗。
4. **状态真实**：未选择、加载中、无差异、二进制、失败和已加载是不同状态，禁止用同一个转圈占位。
5. **稳定布局**：异步状态、计数、按钮禁用和提示文本不得导致主区域尺寸跳动。
6. **平台一致**：优先使用 macOS 标准侧栏、菜单、检查器、工具提示、键盘焦点和语义色。
7. **AI 退居辅助**：AI 不自动运行、不自动提交、不成为完成基础任务的前置条件。

## 4. 目标信息架构

```text
NavigationSplitView
├── Working Copy Sidebar (220...320 pt)
│   ├── 标题 + 添加/移除
│   └── 名称 / 本地路径 / revision / 状态
└── Working Copy Detail
    ├── Context Bar (固定高度)
    │   ├── 当前工作副本摘要
    │   ├── 模式切换
    │   └── 高级功能与全局工具菜单
    └── Changes Workspace
        ├── Main Area
        │   ├── Changes Pane (340...460 pt)
        │   └── Diff Pane (剩余空间)
        └── Commit Inspector (折叠 44 pt / 展开 190...260 pt)
```

变更工作区继续使用固定 `HStack` / `VStack` 和明确 `frame` 约束。不得在该组合层重新引入 `VSplitView` 或 `HSplitView`，避免已知 AttributeGraph 死循环风险。

## 5. 壳层与侧栏

### 5.1 侧栏

- 侧栏默认宽度 `252 pt`，最小 `220 pt`，最大 `320 pt`；
- “工作副本”标题永不换行；添加和移除使用 SF Symbols 图标按钮，并提供 tooltip 与无障碍标签；
- 每行第一行只显示名称和无效状态，名称单行尾部截断；
- 第二行显示缩写后的本地路径，保留 tooltip 展示完整路径；
- 第三行只显示 revision 与简化仓库主机/仓库名，不直接铺满完整 URL；完整 URL 放 tooltip 或上下文菜单；
- 行高稳定，不因“无效”、revision 缺失或长 URL 改变；
- 选中态必须明显，键盘上下键可切换，右键菜单提供移除与在 Finder 中显示。

### 5.2 上下文栏

- 高度固定在标准紧凑工具栏范围，不再使用左右两组互相挤压的自由布局；
- 当前工作副本名称是第一视觉信号，revision 和路径是次级摘要；
- 五个主模式继续使用 segmented control；
- “更多功能”和“工具”改为图标菜单，菜单项保留图标和完整中文名称；
- AI 助手位于工具菜单，不出现在日常模式的固定层；
- `1180 pt` 宽度下所有模式文本完整显示，不允许 segmented item 变成省略号。

## 6. 共享选择与数据流

工作区父层维护唯一的交互选择状态：

```text
WorkspaceSelectionState
├── selectedPaths: Set<String>      // 当前多选，也是默认提交集合
├── focusedPath: String?            // Diff 正在展示的单一文件
└── commitSelectionWasEdited: Bool  // 用户是否主动调整过提交集合
```

数据流规则：

1. 变更列表的单选或多选写入 `selectedPaths`，最近获得焦点的行写入 `focusedPath`；
2. Diff 只读取 `focusedPath`，快速切换时取消或丢弃旧请求结果；
3. 提交检查器默认同步 `selectedPaths`；用户在提交区主动增删候选后，设置 `commitSelectionWasEdited`，避免后续只切换 Diff 文件时意外覆盖提交集合；
4. 刷新状态后移除已不存在的路径，并保留仍有效的选择；
5. 深链和历史页发起 Diff 时，原子设置工作副本、模式、选择路径与 revision 上下文。

共享状态只负责 UI 选择，不复制 `ChangesViewModel`、`DiffViewModel` 或 `CommitViewModel` 的业务职责。

## 7. 变更面板

### 7.1 顶部结构

变更面板拆成两行稳定工具条：

- 第一行：标题、变更计数、刷新、远端检查；
- 第二行：筛选 segmented control、视图 segmented control、搜索、列菜单；
- 搜索框可收缩，但筛选和视图标签不得换行；
- 刷新与远端检查使用图标按钮，tooltip 明确区分“刷新本地状态”和“检查仓库状态”。

### 7.2 文件操作

固定展示的高频操作为：更新、添加、还原。删除使用带下拉的破坏性菜单。清理、重命名、忽略、复制/移动、变更列表、深度和其他完整能力进入“更多操作”菜单。

菜单按“工作副本操作”“所选文件操作”“高级操作”分组，并根据当前选择提供禁用原因 tooltip。右键菜单与顶部菜单共用同一动作定义，避免能力分叉。

### 7.3 列表行为

- 路径列始终存在并占据剩余宽度；状态列固定窄宽；其他列按配置显示；
- 状态颜色只辅助识别，必须同时保留图标或文字；
- 行选中不等于执行动作，破坏性操作始终二次确认；
- 空工作副本显示“没有本地变更”，并保留“刷新”和“检查仓库”动作；
- 加载时保留工具栏与列表框架，使用覆盖式或行内进度，不替换整个布局。

## 8. Diff 面板状态机

Diff 内容区必须显式区分以下状态：

| 状态 | 呈现 | 可用操作 |
|------|------|----------|
| `noSelection` | “选择一个文件查看差异”空态 | 无 |
| `loading(path)` | 保留文件标题，显示小型进度 | 可切换其他文件 |
| `loaded` | Unified 或左右分栏内容 | 模式、BASE、URL、外部工具、刷新 |
| `noChanges` | “此文件没有可显示的文本差异” | BASE、URL、外部工具 |
| `binary` | 二进制说明与文件元信息 | 外部工具 |
| `tooLarge` | 截断说明与规模信息 | 外部工具、继续查看截断内容 |
| `error` | 可读错误、受影响路径、重试 | 重试、外部工具（可用时） |

Diff 工具栏只固定展示模式切换、对比 BASE、外部查看和刷新；“与 URL 比较”等低频能力进入更多菜单。工具栏高度固定，状态文案进入内容区，不新增可变高度提示行。

Unified 和左右分栏继续使用 `DiffPerformanceLimits`。不得为了视觉效果改回逐行渲染超大 Diff。

## 9. 提交检查器

### 9.1 折叠态

- 固定高度 `44 pt`，显示展开图标、“提交”、已选文件数、说明状态和唯一强调按钮；
- 未填写说明或未选择文件时，提交按钮禁用并提供可读原因；
- 点击标题区或展开图标展开，不能让整条区域都误触提交。

### 9.2 展开态

- 高度限制在 `190...260 pt`，默认优先给 Diff 空间；
- 左侧是提交候选摘要与选择入口，右侧是说明编辑器、校验摘要和提交按钮；
- 不再在嵌入模式重复显示大标题和完整候选列表；候选详情通过紧凑 popover 或可滚动区域查看；
- 提交说明编辑器获得明确焦点边界，支持历史说明菜单；
- 本地 Guard、项目属性和冲突校验汇总为一条状态摘要，展开后查看详情；
- 成功后在检查器内显示 revision，刷新变更并保留稳定成功反馈；失败时保留用户输入与选择。

### 9.3 AI 辅助

- “AI 生成说明”和“AI 预检”进入说明编辑器的辅助菜单；
- AI 未配置时不展示无效主按钮，菜单中提供配置入口；
- 生成结果只填入编辑器，由用户确认；预检结果放入校验详情，不改变本地 Guard 的阻断语义；
- AI 请求期间不阻塞手动编辑和普通提交能力。

## 10. 响应式与尺寸标准

应用继续支持最小窗口 `980 x 640`，重点验收窗口为 `1180 x 760`、`1440 x 900` 和全屏。

### 10.1 `1180 x 760` 及以上

- 侧栏、变更面板和 Diff 同时可见；
- 所有主模式、高频操作和提交按钮显示完整名称；
- 不出现只剩 `...` 的按钮，不出现工具栏文字换行；
- 展开提交检查器后，Diff 仍保留可用阅读高度。

### 10.2 `980...1179 pt` 紧凑宽度

- 侧栏保持最小宽度；
- 上下文栏的次级仓库摘要隐藏，完整信息仍可通过 tooltip 获取；
- 变更工具使用图标和 tooltip，低频动作继续进入菜单；
- 不隐藏业务能力，不通过横向滚动工具栏补救；
- 内容无法继续压缩时，由窗口最小尺寸阻止重叠。

## 11. 视觉、动效与无障碍

- 使用系统背景层级、分隔线和 selection material，不给页面区块套装饰卡片；
- 主色只用于选中、焦点和提交按钮；成功、警告、错误使用系统语义色；
- 正文使用系统 body/callout，工具条使用 callout/caption，Diff 使用等宽字体；
- 文本字距保持系统默认，不使用 viewport 驱动字号；
- 图标按钮至少 `28 x 28 pt`，所有图标提供 tooltip 和 accessibility label；
- 关键工作流可通过键盘完成：切换工作副本、上下移动文件、扩大选择、聚焦搜索、展开提交、聚焦说明、提交；
- 动画仅用于检查器展开和状态过渡，时长 `0.16...0.22 s`，尊重 Reduce Motion；
- VoiceOver 顺序遵循侧栏 → 上下文栏 → 变更 → Diff → 提交，不朗读纯装饰图标。

## 12. 错误与进度反馈

- 短操作在触发控件内显示忙碌/禁用状态；长操作使用固定位置状态摘要；
- 错误包含“发生了什么、影响什么、下一步做什么”，保留原始 SVN 错误用于详情；
- 成功反馈不使用抢焦点弹窗，破坏性确认和认证请求除外；
- Update 产生冲突后，明确提供“查看冲突”；
- 提交 out-of-date 错误提供“先更新”动作，但不得自动更新或丢失说明；
- 异步请求切换工作副本或路径时，旧结果不得覆盖新上下文。

## 13. 验收与测试

### 13.1 自动化

- 为共享选择规则、响应式动作分组、Diff 展示状态和提交检查器策略增加纯逻辑单测；
- 保留并扩展 `WorkingCopyWorkspacePerformanceGuardTests`，确认工作区组合层无 SplitView；
- 运行 `swift test` 全量测试；
- 运行应用构建和打包校验，确保 `dist/SVNStudio.app` 可启动。

### 13.2 真实窗口

每个实现切片都必须重新构建并在真实 App 检查：

1. `980 x 640`：无重叠、无错乱换行、能力可达；
2. `1180 x 760`：无按钮省略、三块主内容可读、提交区不抢占；
3. `1440 x 900`：空间被内容有效利用，不出现无意义大空白；
4. 空工作副本、加载、选择文件、快速切换、二进制、错误、提交展开/折叠；
5. 浅色、深色、Reduce Motion、键盘焦点与 VoiceOver 标签。

截图只能证明视觉基线，交互还必须通过真实点击、键盘操作和状态变化验证。

## 14. 完成定义

U5 只有同时满足以下条件才完成：

- 真实窗口完成上述尺寸和状态验收；
- 日常路径“选择 WC → 浏览变更 → 逐文件看 Diff → 选择提交集合 → 填写说明 → 提交”无需进入无关页面；
- AI 不再占据主操作层级，关闭或未配置 AI 不影响基础工作流；
- 现有 SVN 能力、深链和命令面板无回归；
- 全量测试、构建、打包验证通过；
- 无 AttributeGraph 卡死、明显布局跳动、按钮退化为省略号或关键文本重叠。
