# SVN Studio 人本辅助工作流 UI/UX 设计

| 项 | 内容 |
|----|------|
| 日期 | 2026-07-15 |
| 产品 | SVN Studio |
| 状态 | 已批准（沿用用户对 Human UI 后续设计决策的自主授权） |
| 对应迭代 | Human UI Wave U7 |

## 1. 背景

U5 已重排高频变更工作区，U6 已统一历史、浏览、分支、冲突与 Merge。属性、锁、搁置、设置及其弹窗仍保留功能对标阶段的页面结构：

- 属性、锁和搁置重复显示大标题，并继续使用自由 `HSplitView`；
- 目标路径直接显示完整绝对路径，长路径换行后会改变行高和主栏宽度；
- 属性信息、属性列表和编辑器争抢垂直空间，`1180 x 760` 首屏无法稳定完成“选目标 → 读属性 → 编辑 → 保存”；
- 锁列表没有稳定空态或选择详情，获取、释放、打断同时占据固定主层级；
- 搁置把创建参数、官方 shelf、本地快照、Patch 和每行全部动作同时铺开，用户难以区分创建、查看和恢复；
- 设置虽覆盖完整能力，但九类设置缺少搜索、脏状态、保存中状态和可定位的校验错误；
- 进度、成功和错误多为无语义的动态文本，原始 SVN 诊断与用户下一步没有稳定层级。

真实基线截图为 `artifacts/ui/u7-baseline-properties-1180x760.png`。它显示属性页大标题占用首屏、绝对路径换行、自由分栏和编辑器被截断的问题。

U7 不改变 SVN 命令、确认门控、深链、Tortoise inventory 或已有数据模型。它只重构真人完成辅助任务时的呈现和交互顺序。

## 2. 方案选择

### 方案 A：逐页压缩控件

只缩小标题、间距和按钮。改动较少，但属性、锁、搁置继续各自维护不同布局和状态文本，设置也无法获得一致保存反馈。

### 方案 B：共享辅助工作区语言（采用）

新增纯展示策略和共享状态反馈，三页统一为“上下文工具栏 → 目标主栏 → 任务详情”；设置沿用系统侧栏，但共享同一反馈语义。每页保留独立 ViewModel 和业务动作，避免把属性、锁和 shelf 状态塞入一个巨型控制器。

### 方案 C：全部改为独立窗口或系统 Inspector

长期可能适合重度用户，但会改变导航、窗口所有权、深链和状态恢复，超出 U7 的可控范围。

## 3. 统一设计语言

属性、锁和搁置采用相同骨架：

```text
Auxiliary Workflow
├── Context Toolbar (48 pt)
│   ├── 标题 / 当前目标或计数
│   ├── 唯一高频动作
│   └── 刷新 / 低频菜单
├── Inline Feedback (0 或 30 pt，位置稳定)
└── Master Detail Workspace
    ├── Target Master (280...340 pt)
    └── Task Detail (剩余空间，最小 420 pt)
```

规则：

1. 不重复工作副本大标题；壳层已经显示当前工作副本。
2. 目标选择只改变详情，不执行写操作。
3. 一个页面最多一个强调按钮；破坏性动作只通过明确菜单或确认执行。
4. 主组合层使用固定 `HStack` / `VStack`，不使用自由 SplitView。
5. 全路径单行中部截断并提供 tooltip；工作副本根显示为“工作副本根目录”。
6. 加载、空态、错误和成功不替换工具栏，不让内容区域跳动。
7. 小尺寸不删除能力；低频动作进入语义菜单或 sheet。

## 4. 共享展示策略

新增 `MacSvnAuxiliaryWorkflowPresentation.swift`，包含：

```swift
enum MacSvnAuxiliaryWorkflowMetrics {
    static let toolbarHeight: CGFloat = 48
    static let masterWidth: CGFloat = 300
    static let masterMinimumWidth: CGFloat = 280
    static let masterMaximumWidth: CGFloat = 340
    static let detailMinimumWidth: CGFloat = 420
    static let feedbackHeight: CGFloat = 30
}

enum MacSvnAuxiliaryPathPresentation {
    static func relativePath(_ path: String, workingCopy: URL) -> String
    static func title(for path: String) -> String
}
```

路径转换只接受工作副本根或其子路径；工作副本外路径保持原值，禁止用字符串前缀误判相邻目录。

共享反馈分为 `progress`、`success`、`warning`、`failure` 四种语义，展示图标、用户可读摘要和可选重试/下一步动作。原始 SVN 诊断放 tooltip 或详情，不在主界面铺满命令输出。

## 5. 属性工作流

### 5.1 目标主栏

- 顶部显示目标搜索和结果数；
- 路径使用单行中部截断，完整路径通过 tooltip 查看；
- Finder/深链传入绝对路径时先转换为当前工作副本相对路径；
- 单选只触发带 generation 校验的信息和属性加载，不执行写操作。

### 5.2 属性详情

按稳定顺序显示：

1. 目标摘要：修订、作者、状态、锁和仓库 URL；
2. 属性列表：名称、紧凑值摘要、属性数量和明确空态；
3. 属性编辑器：模板、名称、值、诊断、保存；
4. 低频动作：外部定义和带项目模板的提交入口。

属性列表选择会填充编辑器，但不会自动保存。保存是详情区唯一强调按钮；删除只在所选属性菜单中出现并继续二次确认。编辑器可滚动，`980 x 640` 不得把保存按钮挤出可达区域。

## 6. 锁工作流

- 目标主栏支持多选、搜索和“已选 N 项”；
- 详情区优先显示当前锁记录，区分“本工作副本持有”“仓库中由他人持有”“未锁定”；
- “获取锁”是有可锁目标时的主操作；“释放锁”和“打断锁”按当前锁资格进入动作菜单；
- 列表行显示路径、所有者、创建时间和注释摘要，完整值保留 tooltip；
- 获取锁 sheet 保留项目属性模板、最小说明长度、目录目标和夺锁确认；
- 加载失败、没有锁、选中路径不可释放/打断分别显示不同反馈，不使用空白列表表达状态。

## 7. 搁置与 Patch 工作流

- 顶栏固定显示“新建搁置”和刷新；Patch 进入低频菜单；
- 创建参数进入 sheet，先选路径，再填写名称、说明和保留本地改动；
- 主栏按“官方 Shelves / 本地快照”切换，行只显示名称、版本/类型、路径数和时间；
- 选择 shelf 后在详情区展示 Diff、Log 或 Patch 预览，文本继续经过 `DiffPerformanceLimits`；
- Unshelve 是官方 shelf 主操作，恢复是本地快照主操作；Drop、删除、迁移和 Unshelve + Drop 进入动作菜单并保留确认；
- 官方 shelving 不可用时仍保留本地快照与 Patch 能力，说明原因但不将整页置灰。

## 8. 设置工作流

- 保留九类 Tortoise 对标设置和系统侧栏，不改变持久化映射；
- 侧栏增加搜索，按分类标题和设置关键字过滤；无匹配时显示明确空态；
- 内容顶部显示分类标题，底部操作栏固定展示“未保存更改 / 保存中 / 已保存 / 保存失败”；
- 初次加载建立基线快照；字段变化后标记脏状态，保存成功后更新基线；
- 保存按钮仅在有更改且不在保存时启用；保存失败保留全部草稿；
- 校验错误指出分类与具体字段，并提供跳转到对应分类；
- hooks、外置工具规则和 Finder 命令等重复项使用稳定列表行及图标删除按钮，动态增删不改变底部操作栏位置。

## 9. 弹窗、确认与关闭

- 所有自定义 sheet/popover 继续使用共享右上角关闭栏，保留 Esc、tooltip 和 VoiceOver；
- sheet 内保留显式“取消”和默认动作，忙碌时禁止重复提交；
- 关闭包含未保存输入的 sheet 时请求确认，纯查看弹窗直接关闭；
- confirmation dialog 和 alert 继续要求显式 cancel；
- 认证、破坏性远端写和夺锁/打断锁可以抢焦点，其余成功反馈不得弹窗。

## 10. 进度与错误反馈

每条反馈回答三个问题：发生了什么、影响哪个目标、用户下一步能做什么。

- 短操作在触发按钮内显示 busy；
- 页面级加载在固定反馈栏或内容骨架中展示；
- 成功反馈自动保留足够阅读时间，但不抢焦点；
- 可恢复错误提供重试；认证、SSL、网络和超时复用 U6 的可读摘要；
- 切换工作副本或目标后，旧请求结果不得覆盖新上下文；
- 批量操作如有部分成功，必须同时显示成功数、失败数和失败目标。

## 11. 响应式、键盘与无障碍

### `1180 x 760` 及以上

- 主栏与详情同时可读；
- 属性保存、锁主操作和 shelf 恢复不需要滚动整页寻找；
- 设置侧栏与内容不重叠，底部保存栏固定。

### `980...1179 pt`

- 主栏保持至少 `280 pt`，详情至少 `420 pt`；
- 工具栏低频动作收进菜单；
- 路径、URL、锁所有者和 shelf 名称保持单行截断；
- 不出现横向工具栏滚动或自由分栏压缩。

### 键盘与无障碍

- `Command-F` 聚焦当前页面搜索，`Command-R` 刷新；
- 方向键移动目标/记录，Return 进入默认详情动作但不直接执行破坏性操作；
- 保存使用默认动作，Escape 关闭 sheet；
- VoiceOver 顺序为工具栏 → 反馈 → 主栏 → 详情 → 主操作；
- 图标按钮至少 `28 x 28 pt`，具备 tooltip、accessibility label 和稳定尺寸；
- 状态不只依靠颜色，Reduce Motion 下禁用非必要过渡动画。

## 12. 测试与真实窗口验收

### 自动化

- 新增辅助工作流展示策略、路径边界和反馈语义单测；
- 新增属性、锁、搁置和设置源码契约测试；
- 扩展性能守卫，禁止三页主组合层使用 `HSplitView` / `VSplitView`；
- 保留全部 Property、Lock、Shelve、Patch、Settings 业务单测和 49 项真实 SVN 集成；
- 运行 Localization、全量 `swift test`、Release App 构建、结构校验和隔离启动冒烟。

### 真实窗口

属性、锁、搁置和设置分别检查：

1. `980 x 640`、`1180 x 760`、`1440 x 900`；
2. 浅色、深色和 Reduce Motion；
3. 未选择、加载、空态、已有选择、错误、成功和 busy；
4. 搜索、键盘选择、菜单、sheet、取消、默认动作和 Escape；
5. 长路径、长 URL、长属性值、长锁注释和长 shelf 名称。

截图保存为本地 `artifacts/ui/u7-*.png`，不进入 Git 历史。截图只证明视觉基线；写操作、确认、旧请求隔离和错误恢复必须有自动化或真实交互证据。

## 13. 完成定义

U7 只有同时满足以下条件才完成：

- 属性、锁、搁置和设置使用一致、稳定且可学习的交互语言；
- 三档窗口无重叠、关键动作不可达、路径失控换行、工具栏横向滚动或自由 SplitView；
- 所有原有 SVN、Patch、官方/本地 shelf、Finder、设置和深链能力无回归；
- 自定义弹窗均可见关闭、可 Escape、可取消，破坏性动作继续确认；
- 进度、成功、警告和错误具有明确语义，原始诊断仍可访问；
- 全量测试、真实 SVN、Localization、Release 构建、结构校验、隔离冒烟和真实窗口验收通过；
- U8 全局键盘流、无障碍、动效、性能、视觉一致性和真人任务验收仍作为下一波次，不把 U7 完成误报为整个 Human UI 长程目标完成。
