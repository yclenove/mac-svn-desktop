# SVN Studio 人本核心模式 UI/UX 设计

| 项 | 内容 |
|----|------|
| 日期 | 2026-07-15 |
| 产品 | SVN Studio |
| 状态 | 已批准（沿用用户对 Human UI 后续设计决策的自主授权） |
| 对应迭代 | Human UI Wave U6 |

## 1. 背景

U5 已把“变更 → Diff → 提交”整理为稳定的真人高频工作区，但四个主模式仍保留功能堆叠时期的页面结构：

- 历史页把 AI Release Notes、刷新、Next、Show All、统计、三个过滤框和两个开关同时固定展示，窄宽时缺少优先级；
- 仓库浏览器在主窗口内强制收藏、目录、详情三栏，`980 pt` 宽度下三栏最小宽度总和超过可用空间；
- 分支页使用自由 `HSplitView` 并把创建、切换、合并表单常驻，用户难以判断当前选择与主操作；
- 冲突页同时堆叠大标题、模式切换、类型筛选、搜索、批量操作和自由分栏，文本/树/属性三种详情缺少统一阅读顺序；
- Merge 向导仍像设置表单，dry-run、Diff、执行和结果摘要没有稳定的阶段层级。

U6 的目标不是改变 SVN 语义，而是让历史追溯、仓库浏览、分支操作和冲突处理拥有同一套可学习的交互语言。

## 2. 方案选择

### 方案 A：逐页压缩现有控件

保留所有 SplitView 和页面结构，只缩小字体、按钮和间距。改动最少，但只能延后溢出，四页仍然各自为政，用户无法迁移操作习惯。

### 方案 B：统一响应式主从工作区（采用）

四页统一为“上下文工具栏 → 筛选/状态栏 → 主列表 → 详情检查器”。宽窗口并排显示列表与详情；紧凑窗口折叠次级栏和低频动作，但通过菜单、popover 或详情切换保留全部能力。该方案可沿用现有 ViewModel 和命令语义，风险与收益平衡最好。

### 方案 C：多窗口文档式重构

把日志、仓库、分支和冲突拆成独立窗口及系统 Inspector。长期可能更符合重度 macOS 工作流，但会重写导航、深链和状态所有权，超出当前 U6 的可控范围。

## 3. 统一设计语言

四个核心模式遵循同一套结构：

```text
Core Mode
├── Context Toolbar (44...50 pt)
│   ├── 模式标题 / 计数 / 当前上下文
│   ├── 唯一高频动作
│   └── 刷新 / 加载 / 更多菜单
├── Filter or Status Bar (最多 40 pt，可为空)
└── Master Detail Workspace
    ├── Master List (320...400 pt)
    └── Detail Inspector (剩余空间，最小 360 pt)
```

统一规则：

1. 页面内部不再重复工作副本大标题；全局壳层已经提供工作副本上下文。
2. 行选择只改变详情，不直接执行写操作；危险操作必须由明确按钮、菜单或确认触发。
3. 高频动作固定展示，低频动作进入语义菜单；AI 只进入辅助菜单。
4. 加载、空态、错误、成功和离线状态占据稳定区域，不替换整套工具栏。
5. 四页组合层使用确定尺寸的 `HStack` / `VStack` 和宽度策略，不继续依赖自由 `HSplitView`。
6. 图标按钮至少 `28 x 28 pt`，具备 tooltip、accessibility label；菜单隐藏游离指示器。
7. `980 pt` 紧凑宽度不删除能力，不用横向滚动工具栏补救。

## 4. 共享展示策略

新增纯展示策略类型，集中定义可测试的宽度和动作层级：

```swift
enum MacSvnCoreModeWidthClass: Equatable {
    case compact
    case regular

    static func resolve(width: CGFloat) -> Self {
        width < 1_180 ? .compact : .regular
    }
}

enum MacSvnCoreModeMetrics {
    static let toolbarHeight: CGFloat = 48
    static let masterMinimumWidth: CGFloat = 320
    static let masterIdealWidth: CGFloat = 360
    static let masterMaximumWidth: CGFloat = 400
    static let inspectorMinimumWidth: CGFloat = 360
}
```

该策略只决定 UI 呈现，不复制 Log、Repo Browser、Branch、Conflict 或 Merge ViewModel 的业务状态。

## 5. 历史模式

### 5.1 工具栏

- 固定显示“历史”、当前过滤结果数量、刷新图标和加载菜单；
- 加载菜单保留 Next 与 Show All，并显示是否还有更多；
- 统计、复制过滤结果、AI Release Notes 进入“更多历史操作”；
- AI Release Notes 仍带入当前过滤结果，但不占主工具栏。

### 5.2 过滤

- 说明关键字作为最常用搜索框直接展示；
- 作者、路径、Stop on copy、离线模式进入“筛选”popover；
- popover 关闭后，在工具栏显示活动筛选数量和数据源（在线/离线缓存/回退缓存）；
- 三个过滤条件继续可组合，禁止把它们降级为互斥的单范围搜索。

### 5.3 主从内容

- 修订列表宽度稳定在 `320...400 pt`，显示 revision、作者、时间、两行说明和动作摘要；
- 详情按“修订摘要 → 提交说明 → 变更路径 → 操作”阅读；
- 变更路径保持单行中部截断，完整路径放 tooltip；
- 详情只固定展示“查看 Diff”，更新到修订、修订属性、编辑作者/说明、复制摘要进入修订操作菜单；
- 右键 L01–L20 和现有深链能力全部保留。

## 6. 仓库浏览模式

### 6.1 地址与导航

- 顶栏显示可编辑仓库 URL、前往按钮、收藏菜单、刷新和“仓库操作”菜单；
- 收藏不再永久占一整栏，紧凑宽度始终通过菜单访问；
- 目录单击只选择并显示详情，双击或明确“打开”进入目录，避免选择与导航混在一个点击中；
- 当前 URL、所选远端条目和预览请求必须防止旧异步结果覆盖新选择。

### 6.2 目录与详情

- 主列表优先保证名称可读；锁和 revision 使用窄列，长锁持有者通过 tooltip 展示；
- regular 宽度显示右侧详情检查器；compact 宽度通过“详情”按钮打开 popover，不挤压目录列表；
- 详情保留作者、锁信息、检出深度、修订、ignore externals、Checkout 和文本预览；
- 预览明确区分未选择、目录、加载、文本、二进制/不可预览和错误。

### 6.3 远端写

- “新建目录”直接展示；删除、复制、移动、重命名进入“所选条目操作”；
- Export、Import、Import in Place、Relocate、Remove from Version Control 和 Create Repository 保留在“仓库操作”；
- 远端写继续要求提交说明，高危确认和目标快照语义不变。

## 7. 分支模式

- 顶栏显示“分支与标签”、分支/标签数量、刷新和“创建分支/标签”；
- 主列表按主干、分支、标签提供 segmented filter，行显示名称、简化 URL 和 revision；
- 选择项详情按“引用摘要 → 当前工作副本关系 → 可执行动作”组织；
- “切换到此引用”是选择已有引用后的主操作；查看日志、复制 URL、检出进入更多操作；
- 创建分支/标签使用现有表单能力，但放入弹窗，不和选择详情常驻竞争空间；
- 合并入口进入专用 Merge 向导，并自动带入所选引用 URL；Switch、两树 Merge、reintegrate 与 dry-run 能力不变。

## 8. 冲突与 Merge 模式

### 8.1 冲突列表

- 顶栏显示冲突总数、冲突列表/Merge 向导 segmented、刷新和返回变更；
- 类型筛选、路径搜索和活动数量放在单行稳定过滤栏；
- 复选框只控制批量标记集合，行选择只控制右侧详情；
- “全选可解决”“清除勾选”进入选择菜单，“标记已解决”作为唯一批量主命令；
- 文本、树、属性和未知冲突同时保留图标/文字，不能只靠颜色区分。

### 8.2 冲突详情

- 详情顶部始终显示冲突类型、完整路径、状态和可用主操作；
- 文本合并按 Mine / Base / Theirs → Result 的阅读顺序，AI 建议和外置 Merge 进入辅助菜单；
- 树冲突明确展示 operation/action/reason，并将“保留本地/接受远端”作为互斥命令；
- 属性冲突保留 Mine/Base/Theirs 与接受策略；
- 解决成功后刷新列表并稳定选择下一项，不返回无上下文空白页。

### 8.3 Merge 向导

- 将合并类型、来源/目标、revision 范围放入紧凑参数区；
- “Dry-run 预览”固定展示，“Unified Diff”按能力启用，“执行合并”是唯一强调按钮；
- 预览、Diff 和执行共享固定结果区，摘要先于路径明细；
- 执行产生冲突时原子跳回冲突列表并预选首个冲突；
- `DiffPerformanceLimits` 与现有真实 SVN 语义保持不变。

## 9. 响应式标准

### `1180 x 760` 及以上

- 主列表与详情同时可读；
- 工具栏无文字按钮省略、游离菜单指示器或多行换行；
- 仓库浏览不再固定三栏，详情至少保留 `360 pt`；
- Merge 结果区不把参数区挤出首屏。

### `980...1179 pt`

- 主列表最小宽度 `320 pt`，详情获得剩余空间；
- 仓库收藏与详情进入菜单/popover；
- 说明搜索保留，历史高级筛选进入 popover；
- 分支创建和低频操作进入菜单/弹窗；
- 冲突批量选择操作进入菜单，标记已解决保持可达；
- 不隐藏任何 SVN 能力，不出现横向工具栏滚动。

## 10. 状态、键盘与无障碍

- `⌘F` 聚焦当前模式搜索；`⌘R` 刷新；方向键移动主列表；Return 打开目录或默认详情动作，但不得直接执行破坏性写操作；
- 加载时保留列表骨架和当前选择；错误提供重试且不清空用户输入；
- VoiceOver 顺序为模式工具栏 → 筛选 → 主列表 → 详情 → 主操作；
- 选中、冲突、锁和数据源状态使用文字/图标/颜色组合；
- 动画只用于详情切换和 popover，尊重 Reduce Motion；
- 所有异步详情请求以当前工作副本、URL、revision 或 path 为上下文校验，旧结果不得越界覆盖。

## 11. 测试与验收

### 自动化

- 新增 U6 展示策略和源码契约测试；
- 保留 Log、Repo Browser、Branch、Conflict、Merge 业务单测与真实 SVN 集成测试；
- 扩展性能守卫，禁止 U6 主组合层继续使用自由 SplitView；
- 运行 Localization、全量 `swift test`、Release 构建与 App 冒烟。

### 真实窗口

对历史、仓库浏览、分支、冲突列表和 Merge 向导分别检查：

1. `980 x 640` 内容区；
2. `1180 x 760` 默认窗口；
3. `1440 x 900` 宽窗口；
4. 浅色、深色、加载、空态、错误和已选详情；
5. 搜索、行选择、详情切换、菜单、弹窗和 Escape 关闭；
6. 键盘焦点、tooltip 和 VoiceOver 标签。

截图只作为视觉证据；选择、导航、写操作确认和旧请求隔离必须有自动化或真实交互证据。

## 12. 完成定义

U6 只有同时满足以下条件才完成：

- 四个核心模式使用一致的上下文栏、筛选、主列表和详情语义；
- `980`、`1180`、`1440` 三档无重叠、截断失控、工具栏换行或横向滚动；
- 历史组合筛选、L01–L20、仓库远端写/传输、分支创建/切换/合并、三类冲突与批量解决能力无回归；
- AI 仅作为辅助入口，未配置 AI 不影响任何基础流程；
- 全量测试、真实 SVN 集成、Localization、Release 构建、App 冒烟和真实窗口验收通过；
- U7/U8 仍作为后续波次，不把 U6 完成误报为整个 Human UI 长程目标完成。
