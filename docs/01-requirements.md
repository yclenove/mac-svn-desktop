## 1. 引言

### 1.1 编写目的

本文档定义 MacSVN 的完整产品需求，作为需求分析、概要设计、详细设计、测试计划的唯一上游依据。读者对象：开发、测试、后续贡献者。

### 1.2 产品定位

面向 macOS 的**开源全功能 Subversion 桌面客户端**，对标商业客户端 Versions / Cornerstone，核心差异化：

1. **内置三路合并编辑器**——文本冲突不依赖外部工具即可解决；
2. **仓库浏览器**——不检出即可浏览远端目录、查看日志、执行浅检出；
3. **中文友好**——UTF-8 提交说明无乱码、中文界面、中文路径可靠。

### 1.3 术语定义

| 术语 | 含义 |
|------|------|
| WC（Working Copy） | 工作副本，`svn checkout` 到本地的目录 |
| Revision（版本号） | SVN 全局递增的提交序号，形如 r1234 |
| 三路合并 | 基于 Base（共同祖先）/ Mine（本地）/ Theirs（远端）三份内容的合并 |
| 浅检出 | 使用 `--depth`（empty/files/immediates/infinity）限制检出深度 |
| 树冲突 | 目录结构级冲突（如本地编辑的文件被远端删除） |
| 属性（Property） | SVN 附加在文件/目录上的元数据，如 `svn:ignore`、`svn:eol-style` |

### 1.4 参考资料

- Apache Subversion 1.14 官方文档（`svn help`、SVN Book）
- 原始决策记录：`docs/specs/2026-07-08-mac-svn-desktop-design.md`

## 2. 总体描述

### 2.1 用户画像

| 角色 | 场景 | 关键诉求 |
|------|------|----------|
| 后端/前端开发者 | 日常提交、看 diff、查历史 | 快、稳定、中文不乱码 |
| 技术负责人 | 团队统一工具 | 免费、可配置、可审计 |
| 非 CLI 用户（策划/美术/文档岗） | 图形化 update/commit | 零命令行、误操作可恢复 |
| 从商业客户端迁移的用户 | 替代 Versions/Cornerstone | 功能覆盖度、操作习惯相近 |

### 2.2 运行环境

| 项 | 要求 |
|----|------|
| 操作系统 | macOS 14.0+（Sonoma 及以上） |
| 依赖 | Subversion CLI ≥ 1.14（Homebrew 或 Xcode CLT 提供） |
| 网络 | 支持 http(s)://、svn://、svn+ssh://、file:// 协议访问仓库 |
| 硬件 | Apple Silicon 与 Intel 双架构（Universal Binary） |

### 2.3 设计与实现约束

1. 底层通过 `svn` CLI（`--xml` 输出）实现，通过 `SvnBackend` 协议预留 libsvn 替换点；
2. 密码不得出现在进程参数（argv）中，凭据复用 svn 自身 Keychain 缓存；
3. 所有子进程强制 `LC_ALL=C` 运行，避免本地化输出破坏解析；
4. 应用使用 SwiftUI 构建，非沙箱分发（需要访问任意用户目录与 svn 二进制）。

## 3. 功能需求

需求编号规则：`FR-<模块>-<序号>`。优先级：P1（MVP 必须）> P2 > P3 > P4，与路线图阶段对应。

### 3.1 工作副本管理（WC）

| 编号 | 需求 | 优先级 |
|------|------|--------|
| FR-WC-01 | 用户可通过文件选择器或拖拽添加本地目录为工作副本；系统校验其为合法 WC（含 `.svn`），非法时给出明确提示 | P1 |
| FR-WC-02 | 侧边栏展示已添加的 WC 列表，显示名称、路径、仓库 URL、当前 revision | P1 |
| FR-WC-03 | 可从列表移除 WC 记录（仅移除记录，不删除磁盘文件；需二次确认） | P1 |
| FR-WC-04 | WC 列表持久化保存，应用重启后恢复；路径失效（目录被删）时标记为不可用 | P1 |
| FR-WC-05 | 支持从仓库 URL 直接 Checkout 新 WC（完整检出） | P2 |
| FR-WC-06 | Checkout 支持浅检出（`--depth empty/files/immediates/infinity`），已有 WC 可调整深度（`update --set-depth`） | P2 |
| FR-WC-07 | 支持对 WC 执行 Cleanup（解除锁定、清理中断操作） | P1 |

### 3.2 变更与状态（Status）

| 编号 | 需求 | 优先级 |
|------|------|--------|
| FR-ST-01 | 展示 WC 内全部变更文件，状态含：M（修改）、A（新增）、D（删除）、?（未纳管）、!（缺失）、C（冲突）、R（替换）、外部/忽略项可选显示 | P1 |
| FR-ST-02 | 变更列表支持目录树分组与平铺两种视图，支持按状态筛选、按文件名搜索 | P1 |
| FR-ST-03 | 状态刷新为异步操作，展示进度，可取消；大 WC（10 万文件级）不阻塞 UI | P1 |
| FR-ST-04 | 未纳管文件可执行 `svn add`；纳管文件可执行 `svn delete`、`svn revert`（revert 需二次确认） | P1 |
| FR-ST-05 | 支持忽略配置：将文件/模式加入 `svn:ignore` 属性 | P4 |

### 3.3 更新与提交（Update / Commit）

| 编号 | 需求 | 优先级 |
|------|------|--------|
| FR-UP-01 | 一键 Update 整个 WC 或选中子路径；展示更新进度与结果摘要（更新/新增/删除/冲突计数） | P1 |
| FR-UP-02 | Update 产生冲突时，结果摘要中醒目标记，并可跳转到冲突解决界面 | P1（跳转 P3） |
| FR-CM-01 | 提交对话框：勾选参与提交的文件（默认全选变更文件）、填写提交说明 | P1 |
| FR-CM-02 | 提交说明强制 UTF-8 编码（`--encoding UTF-8`），中文/emoji 不乱码；说明为空时阻止提交 | P1 |
| FR-CM-03 | 提交历史说明可快速复用（最近 10 条本地记录） | P2 |
| FR-CM-04 | 提交前自动检测选中文件中的冲突状态文件并阻止提交 | P1 |
| FR-CM-05 | 提交成功后展示新 revision 号，自动刷新状态 | P1 |

### 3.4 差异对比（Diff）

| 编号 | 需求 | 优先级 |
|------|------|--------|
| FR-DF-01 | 双击变更文件查看 Unified Diff 文本视图（语法着色：新增绿/删除红） | P1 |
| FR-DF-02 | Side-by-side（左右分栏）对比视图，行级对齐、块内字符级高亮 | P4 |
| FR-DF-03 | 支持任意两个 revision 之间的文件 diff（从 Log 界面发起） | P2 |
| FR-DF-04 | 二进制文件提示"无法显示差异"，展示大小/修改时间变化 | P1 |
| FR-DF-05 | 可配置外部 Diff 工具（Kaleidoscope、Beyond Compare 等）作为可选打开方式 | P4 |

### 3.5 日志（Log）

| 编号 | 需求 | 优先级 |
|------|------|--------|
| FR-LG-01 | 展示 WC 或选中路径的提交历史：revision、作者、日期、说明、变更文件列表 | P1 |
| FR-LG-02 | 分页/增量加载（每批 100 条），支持"加载更多" | P1 |
| FR-LG-03 | 按作者、日期范围、说明关键字过滤 | P2 |
| FR-LG-04 | 从日志条目可发起：查看该版本 diff、更新到该版本（`update -r`）、从该版本还原文件 | P2 |
| FR-LG-05 | 远端路径（未检出）的日志查看（仓库浏览器内） | P2 |

### 3.6 仓库浏览器（Repository Browser）

| 编号 | 需求 | 优先级 |
|------|------|--------|
| FR-RB-01 | 输入仓库 URL 后浏览远端目录树（懒加载，展开时请求） | P2 |
| FR-RB-02 | 展示条目元数据：名称、大小、最后修改 revision/作者/时间 | P2 |
| FR-RB-03 | 远端文本文件内容预览（`svn cat`，限制大小上限 5 MB） | P2 |
| FR-RB-04 | 从远端目录发起 Checkout（含浅检出选项） | P2 |
| FR-RB-05 | 远端目录/文件的日志查看 | P2 |
| FR-RB-06 | 远端操作：创建目录（mkdir）、删除、复制、移动（均直接产生服务端提交，需填写提交说明） | P4 |
| FR-RB-07 | 仓库 URL 收藏列表，持久化保存 | P2 |

### 3.7 分支与标签（Branch / Tag）

| 编号 | 需求 | 优先级 |
|------|------|--------|
| FR-BR-01 | 按约定目录结构（trunk/branches/tags，可配置）列出分支与标签 | P2 |
| FR-BR-02 | 创建分支/标签：从 WC 或远端 URL 执行 `svn copy`，填写提交说明 | P2 |
| FR-BR-03 | 切换分支：对 WC 执行 `svn switch`，切换前检测未提交变更并警告 | P2 |
| FR-BR-04 | Merge 向导：选择来源分支与 revision 范围，预览受影响文件（`merge --dry-run`），执行合并 | P2 |
| FR-BR-05 | 展示合并追踪信息（`svn:mergeinfo`，已合并 revision 范围） | P4 |

### 3.8 冲突解决（内置三路合并，核心差异化）

| 编号 | 需求 | 优先级 |
|------|------|--------|
| FR-CF-01 | 集中展示 WC 内全部冲突文件（文本冲突、树冲突、属性冲突分类） | P3 |
| FR-CF-02 | 文本冲突：三窗格合并编辑器——上方 Base / Mine / Theirs 三栏只读对照，下方合并结果可编辑 | P3 |
| FR-CF-03 | 冲突块导航（上一处/下一处），每块支持：采用 Mine / 采用 Theirs / 双方保留（Mine 在前或 Theirs 在前）/ 手动编辑 | P3 |
| FR-CF-04 | 所有冲突块处理完成前，"标记已解决"按钮不可用；完成后写回工作文件并执行 `svn resolve --accept working` | P3 |
| FR-CF-05 | 整文件快捷操作：全部采用 Mine（`--accept mine-full`）/ 全部采用 Theirs（`--accept theirs-full`） | P3 |
| FR-CF-06 | 树冲突：展示冲突原因描述，提供"保留本地/接受远端"选择 | P3 |
| FR-CF-07 | 属性冲突：展示双方属性值，选择保留哪一方 | P4 |
| FR-CF-08 | 合并编辑器内容未保存离开时警告；支持放弃本次合并编辑（不影响冲突状态） | P3 |

### 3.9 Blame / 属性 / 锁定

| 编号 | 需求 | 优先级 |
|------|------|--------|
| FR-BL-01 | Blame 视图：逐行展示最后修改的 revision、作者、时间，点击行可跳转该 revision 日志 | P4 |
| FR-PR-01 | 查看/编辑文件与目录的 SVN 属性（增删改），常用属性提供模板（`svn:ignore`、`svn:eol-style`、`svn:executable`、`svn:needs-lock`） | P4 |
| FR-LK-01 | 获取锁（lock，可填注释）、释放锁（unlock）、强制夺锁（需确认）；列表展示锁定状态与持有者 | P4 |

### 3.10 认证与设置

| 编号 | 需求 | 优先级 |
|------|------|--------|
| FR-AU-01 | 优先复用 svn 自身凭据缓存（macOS Keychain）；无缓存或认证失败时弹出凭据输入框 | P1 |
| FR-AU-02 | 用户输入的密码通过 `--password-from-stdin` 传递，禁止出现在 argv；用户名可随 WC 记录保存 | P1 |
| FR-AU-03 | 支持 svn+ssh 协议（依赖系统 ssh 配置，应用内提示配置方法） | P2 |
| FR-SE-01 | 设置项：svn 可执行路径（自动探测 + 手动指定）、日志加载批量大小、分支目录约定、外部 diff 工具 | P1（外部工具 P4） |
| FR-SE-02 | 首次启动检测 svn 可用性与版本（≥ 1.14），不满足时展示引导页（brew 安装指引） | P1 |

## 4. 非功能需求

| 编号 | 类别 | 需求 |
|------|------|------|
| NFR-01 | 性能 | 1 万文件 WC 的 status 结果在 3 s 内呈现；10 万文件级 WC 不崩溃、不阻塞 UI，可取消 |
| NFR-02 | 性能 | 日志单批（100 条）加载在正常内网环境 2 s 内完成 |
| NFR-03 | 响应性 | 所有 svn 子进程调用均为异步；任何长任务展示进度并可取消 |
| NFR-04 | 安全 | 密码不落盘、不进日志、不上 argv；日志中的 URL 凭据段脱敏 |
| NFR-05 | 可靠性 | 子进程超时保护（默认 120 s，可配置）；应用崩溃不损坏 WC（写操作交由 svn 保证原子性） |
| NFR-06 | 兼容性 | svn 1.14 为基线；1.10–1.13 降级运行（禁用 `--password-from-stdin`，改为交互引导） |
| NFR-07 | 可观测性 | `os.Logger` 分级日志；每次 svn 调用记录命令、耗时、退出码（脱敏后） |
| NFR-08 | 国际化 | 首版中文 UI，文案集中管理（String Catalog），预留英文 |
| NFR-09 | 可维护性 | 解析层与合并引擎 100% 可单测；UI 与业务逻辑分离 |
| NFR-10 | 分发 | P4 阶段完成 Developer ID 签名 + 公证；Universal Binary |

## 5. 明确不做（Out of Scope）

1. SVN 服务器管理（仓库创建、hook、权限配置）；
2. Git-SVN 迁移向导；
3. 内置通用代码编辑器（合并结果区仅提供必要编辑能力）；
4. Windows / Linux 版本；
5. 多语言协议扩展（如 SVN 属性驱动的自定义工作流）。

## 6. 验收标准（按阶段）

| 阶段 | 验收口径 |
|------|----------|
| P1 | 新用户 3 分钟内完成：添加 WC → 查看变更 → 提交中文说明（无乱码）；update/revert/cleanup/log/unified diff 全部可用 |
| P2 | 不检出任意仓库即可浏览目录与日志；完成一次浅检出；完成一次分支创建与切换；merge 向导可执行合并并正确产生冲突 |
| P3 | 构造文本冲突后，全程在应用内完成三路合并并 resolve，`svn status` 确认冲突消除；树冲突可选择保留方 |
| P4 | blame/属性/锁定可用；应用通过公证，可在未安装开发工具的干净 macOS 上运行（svn 依赖除外） |
